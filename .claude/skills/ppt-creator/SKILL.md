# ppt-creator — OI 教学课件生成

> 把讲义变成学生看得见的东西。

## 定位与分工

ppt-creator是**视觉出片层**：

- **备课**技能 → 教学设计、讲义文本（学什么、怎么讲、例题选什么）
- **ppt-creator**技能 → 把讲义出成单文件 HTML 课件（看得见、课堂能放、学生能拿走）

输入：讲义文本（来自备课，或老师直接给的内容）
输出：一个 `.html` 文件，浏览器双击即放，方向键翻页

## 工作目录

- 模板：`resource/ppt-creator/assets/template.html`（必读，出片前先拷贝）
- 布局库：`resource/ppt-creator/references/layouts.md`（OI 教学版式骨架）
- 主题：`resource/ppt-creator/references/themes.md`（课件配色）
- 组件手册：`resource/ppt-creator/references/components.md`（代码块、流程图、表格、对比卡等）
- 自检清单：`resource/ppt-creator/references/checklist.md`

## 出片流程

### 阶段 1：澄清（5 问法）

拿到需求先问清，别急着出：

1. **场景**：算法知识点讲义 / 题目讲解 / 复习串讲 / 课堂教案？
2. **受众**：入门组还是提高组？决定术语密度和假设的前置知识
3. **内容来源**：有备课技能产出的讲义吗，还是现场给内容？
4. **时长/页数**：一节课 40 分钟大概 15-25 页；复习串讲可更密
5. **重点**：这节课最想让学生记住的一件事是什么？

### 阶段 2：搭骨架

按场景选页面顺序（见 layouts.md 的场景模板）：

- **算法知识点讲义**：封面→为什么学→概念定义→图示→代码模板→复杂度→例题→小结
- **题目讲解**：题面→数据范围分析→思路推演→核心代码→边界→变式
- **复习串讲**：知识地图→逐点快览（每点 1-2 页：定义+坑+典型题）→易错合集→练习指向
- **课堂教案**：学习目标→复习引入→新知讲解→例题演练→课堂练习→总结作业（带时间分配）

### 阶段 3：拷模板 + 填内容

1. 读取 `resource/ppt-creator/assets/template.html`
2. 在 `<!-- DECK -->` 区间内，按骨架顺序填入各页
3. 每页从 layouts.md 选一个版式骨架，替换占位内容
4. 代码段用组件手册里的「代码块」组件（带语法高亮、行号、可标注重点行）
5. 算法过程用「流程图/步骤卡」组件，别用大段文字描述流程

### 阶段 4：自检

对照 `resource/ppt-creator/references/checklist.md` 逐条过，P0 级必须全过（字号可读、代码不溢出、对比度达标、关键页有视觉锚点）。

### 阶段 5：交付

输出单个 HTML 文件，文件名格式 `课件-<主题>-<日期>.html`。浏览器打开即可演示，方向键 ← → 翻页。

## OI 课件特有要求

这是教学课件，不是商业 PPT。以下是硬约束：

1. **代码必须高亮且可读**：等宽字体、语法着色、长行折行不截断、关键行能高亮标注。代码是主角，字号不能小于 18px。
2. **算法要可视化**：讲排序就放步骤动画或静态分步图；讲 DP 就放状态转移表/网格；讲图论就放节点边示意图。纯文字讲算法 = 失败课件。
3. **每页一个要点**：一页别塞超过一个概念。宁可多翻一页，不要挤一屏。
4. **数学公式支持**：复杂度、递推式、组合数用公式渲染（模板内置轻量方案），别用纯文本写 `O(n log n)` 之类的凑合。
5. **学生能拿走自学**：课件脱离老师讲解也能看懂七八成。关键页要有"为什么"的注解，不只是结论。
6. **配色克制**：从 themes.md 选预设主题，不要自定义花哨配色。浅底深字为主，投影仪看得清。

## 与备课协作

理想流程：`备课`产出讲义 markdown → `ppt-creator`读取并出片。两技能可串接：

```
用户: 帮我备一节线段树的课，并做成课件
→ 备课: 产出讲义（学习目标、讲解结构、例题、板书要点）
→ ppt-creator: 读讲义，按"算法知识点讲义"骨架出 HTML 课件
```

若用户直接给内容、跳过备课，ppt-creator也能独立运作——但建议先问"要不要先走备课流程梳理教学设计"。

## 风格

遵循 CLAUDE.md。课件是工具活，干净专业为主；但封面和过渡页可以有一点温度（比如开场用一句话点题，结尾页放鼓励而非干巴巴的"谢谢"）。
