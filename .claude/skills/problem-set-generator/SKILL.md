# 题单与训练计划生成器

## 何时使用

当学生提出以下需求时主动触发：
- "给我出一套练习题"
- "我想练练 XXX（知识点）"
- "接下来一周/一个月怎么安排训练"
- "我要参加 XX 比赛，需要准备"
- "我哪里薄弱，帮我补一补"

## 工作流程

### 0. 读取学生画像（如果存在）

**先读 `memory/student/` 下的文件，再生成题单：**

- `profile.md` — 了解学生画像、目标、薄弱点
- `knowledge_matrix.md` — 查看各知识点掌握状态
- `error_log.md` — 查看近期错题和高频错误模式
- `templates.md` — 查看模板熟练度
- `contest_log.md` — 查看最近比赛表现

**如果有历史数据**：基于画像直接生成个性化题单。
**如果是第一次**：进入诊断阶段（步骤 1）。

### 1. 诊断阶段

| 必问项 | 说明 |
|---------|------|
| 当前水平 | 入门 / 基础 / 提高 / 冲刺 |
| 编程语言 | C++ / Python / 其他 |
| 目标比赛 | NOIP普及组 / NOIP提高组 / CSP / 蓝桥杯 |
| 可用时间 | 每天能投入多少小时 |
| 薄弱环节 | 哪些知识点不太熟 |

### 2. 生成题单

```markdown
## 训练题单：{主题名称}

**目标：** {简述训练目的}
**难度梯度：** {入门→提高}
**预计用时：** {X 天 / X 题}

| # | 题目 | 难度 | 知识点 | 来源 | 建议用时 |
|---|------|------|--------|------|---------|
| 1 | [题目名](链接) | ⭐ | 知识点 | 平台 | 5min |
```

**难度标注**：
- ⭐ 入门（语法基础，直接模拟）
- ⭐⭐ 基础（简单数据结构，基础算法）
- ⭐⭐⭐ 提高（经典算法，需要思维转换）
- ⭐⭐⭐⭐ 进阶（综合应用，多算法结合）
- ⭐⭐⭐⭐⭐ 冲刺（比赛真题级别）

**推荐题库来源**：洛谷、Codeforces、AtCoder、LeetCode、LibreOJ

### 3. 生成训练计划

```markdown
## {周期}训练计划

**阶段目标：** {本阶段要达到的能力}
**时间跨度：** {X 周 / X 月}

### 第 1 周：{主题}
| 日期 | 内容 | 题单 | 目标 |
|------|------|------|------|
| Day 1 | 学习 {知识点} | 3-5 题 | 理解概念 |
| Day 2 | 练习 {知识点} | 5-8 题 | 熟练应用 |
| Day 7 | 周总结 + 模拟赛 | 1 场 | 查漏补缺 |
```

**计划原则**：
- 循序渐进：不要一上来就挑战难题
- 刻意练习：每个知识点至少 10-15 题
- 及时反馈：每天记录做对率，每周复盘
- 劳逸结合：每周至少一天休息

### 4. Upsolving 环节（赛后补题）

> 做过的题不复盘，等于白做。比赛完不补题，等于白比。

```markdown
## Upsolving 计划：{比赛名称}

| 题号 | 题目 | 赛中状态 | 补题状态 | 完成日期 |
|------|------|---------|---------|---------|
| A | | ✅做出 | — | — |
| B | | ❌没做出来 | 🟡进行中 | |
```

**Upsolving 规则**：
- 没做出来的题，赛后要独立做出来
- 看题解后，必须自己重新写一遍（不是抄）
- 三天后重做，检验是否真正掌握

### 5. 模板训练计划

```markdown
## 模板训练：{算法名称}

**目标：** 闭眼能写，肌肉记忆

### 训练步骤
1. 理解算法原理（不写代码，先讲清楚思路）
2. 照着模板敲一遍（理解每行代码的作用）
3. 空手写一遍（不看任何参考）
4. 限时写一遍（5 分钟内完成）
5. 变体写一遍（稍作修改的场景）

### 检验标准
- [ ] 能独立写出，无语法错误
- [ ] 5 分钟内完成
- [ ] 能解释每行代码的作用
- [ ] 能做简单变体
```

## 知识点地图

### 入门阶段
变量与数据类型、条件语句与循环、函数与递归基础、一维数组与字符串

### 基础阶段
二维数组与结构体、排序算法、栈与队列、贪心与模拟、二分查找

### 提高阶段
动态规划入门（背包、区间、树形）、图论基础（BFS、DFS、最短路径）、并查集、前缀和与差分

### 进阶阶段
线段树与树状数组、高级 DP（状态压缩、数位 DP）、图论进阶、字符串算法（KMP、Trie）、数学（组合数学、数论）

### 冲刺阶段
网络流、后缀数组/自动机、平衡树、计算几何、博弈论

## 记忆更新

生成题单/计划后，更新学生记忆：

1. **知识矩阵** — 更新涉及的知识点状态
2. **错题本** — 记录学生完成的题目错误
3. **模板库** — 更新模板熟练度
4. **比赛记录** — 记录比赛成绩和 upsolve 进度
5. **画像** — 发现学生新特点时更新
6. **对话记录** — 记录本次生成的题单和学生反馈
