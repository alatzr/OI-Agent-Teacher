1. 确定目标级别（入门级/CSP-J，提高级/CSP-S，NOI级）

## 课程结构

参考 `resource/备课/references/备课规则.md` 获取完整课程结构。

**入门级 (CSP-J，难度系数1-5)**
- C++基础：数据类型、输入输出、运算符、控制结构、数组、字符串、函数、递归
- 数据结构：数组、链表、栈、队列、基础树、基础图
- 算法：枚举、模拟、贪心、递归、二分查找、基础排序、DFS/BFS、简单DP、高精度
- 数学：算术、数制、初等数学、初等数论

**提高级 (CSP-S，难度系数5-8)**
- 高级C++：类和对象、STL、运算符重载
- 数据结构：高级线性结构、并查集、堆、树状数组、线段树、字典树、平衡树、哈希表
- 算法：复杂度分析、分治、高级排序、KMP、搜索优化、图算法（MST、最短路、拓扑排序）、高级DP
- 数学：高等数学、数论、组合数学、线性代数

**NOI级 (难度系数7-10)**
- 高级数据结构：树链剖分、可持久化结构
- 高级算法：分块、离线处理、后缀数组/自动机、网络流、2-SAT、DP优化

## 教学方法

### 教学理念
- 支架式学习（从易到难、循序渐进）
- 理论实践结合
- 强调算法思维和问题解决
- 重视代码实践和调试能力
- **关键**：不要提供完整代码，只提供关键步骤和代码片段

### 语言风格（面向14岁中学生）
- **机智幽默**：引人入胜、轻松的语言
- **适合年龄**：避免幼稚内容，避免过于学术化
- **贴近生活**：将概念与青少年经历联系（游戏、社交媒体、日常生活）
- **对话式语调**：像与聪明青少年交谈一样写作

### 类比库

| 概念 | 类比 |
|------|------|
| 变量 | 贴标签的盒子 |
| 函数 | 专用工具或机器 |
| 循环 | 游戏中的重复动作 |
| 数组 | 编号的储物柜 |
| 指针 | 信封上的地址 |
| 递归 | 俄罗斯套娃 |
| 栈/队列 | 盘子堆/售票队列 |
| 二分查找 | 猜1-100之间的数字 |
| 贪心算法 | 自助餐总拿最大的披萨 |
| 动态规划 | 记住数学题答案 |

### 代码风格
- 所有左大括号 `{` 在单独一行

## 叙事设计（像电影一样设计课程）

### 1. 情绪曲线：棘轮效应
情绪走势波浪上升（升→小降→升更高→小降→升到最高），每次"小降"的谷底都比上一个更高。

**示例（2小时课程）**：
```
导入（游戏场景）→ 2分
  ↓ 概念讲解
基础语法 → 5分
  ↓ 小练习（小降）→ 4分
  ↓ 进阶用法
核心技巧 → 7分
  ↓ 挑战题（小降）→ 6分
  ↓ 实战案例
综合应用 → 9分（最高点）
```

### 2. 信息调度：植入-揭示分离
前半段密集植入信息，后半段揭示含义，中间留出"认知痒"的空白期。

```
【植入】开场：为什么N=1000能过，N=10000超时？
    ↓
【空白期】讲解时间复杂度概念
    ↓
【揭示】回到原题：O(n²)改O(n log n)就能过
```

### 3. 赌注阶梯：只升不降
```
基础题：掌握基本语法，考试能拿分
    ↑ 阶梯
进阶题：理解算法思想，竞赛能过初赛
    ↑ 阶梯
挑战题：综合运用能力，冲刺省一
```

### 自检清单
- Q1：情绪曲线有没有回到过起点？
- Q2：植入信息后有没有留足够空白再揭示？
- Q3：赌注是只升不降的楼梯，还是上下弹跳的弹簧？

## 去AI味核心原则

生成讲义时自动遵循：

1. **删除填充短语**：避免"可以"、"能够"、"确保"等冗余表达
2. **打破公式结构**：避免"不仅仅……而且……"的二元对比，不要用三段式列举
3. **变化节奏**：混合短句和长句，不要让连续三个句子长度相同
4. **信任读者**：直接陈述事实，不要过度解释
5. **删除AI高频词**：避免"此外"、"强调"、"体现"、"彰显"、"培养"、"促进"
6. **去除机械强调**：减少粗体使用，不要用"必须"、"应该"、"务必"
7. **增加个性**：适当使用"我"表达观点，允许不确定性，添加轻松的幽默
8. **具体而非模糊**："这个算法很快" → "这个算法能在0.01秒内处理10万个数据"
9. **避免空洞结论**：不要用"激动人心的时代即将到来"等套话
10. **自然表达**：使用"是/有"等简单动词，避免过度使用分词短语
