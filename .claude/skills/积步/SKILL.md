# 积步 — 不积跬步，无以至千里

> 知识不是听过就算拥有，沉淀下来、定期翻看，才是你的。

## 定位与边界

积步是**记忆系统之上的智能层**：

- 记忆系统（见 `CLAUDE.md` + `memory/student/INDEX.md`）负责**记录**：把对话要点写进各记忆文件
- 积步负责**沉淀 + 检索 + 调度 + 诊断**：把记录提炼成可复习的知识卡片、安排复习时机、给出画像诊断

两者不重复。记忆是原料，积步是把原料炼成可用的东西。

## 工作目录

- 学生记忆：`memory/student/`（真实数据，本地；模板在 `memory/student.example/`）
- 知识卡片沉淀处：`resource/积步/cards/`（按知识点命名，如 `resource/积步/cards/前缀和.md`）
- 复习调度参考：`resource/积步/复习调度.md`
- 卡片模板：`resource/积步/知识卡片模板.md`

## 四个功能

### 1. 沉淀（自动 + `/积步 沉淀`）

当一段关于某个知识点的对话告一段落（学生理解了、或做对/做错了相关题），提炼并沉淀：

1. **更新知识矩阵**：`memory/student/knowledge_matrix.md` 里该知识点的状态
2. **生成/更新知识卡片**：写入 `resource/积步/cards/<知识点>.md`，结构见模板：
   - 一句话定义
   - 学生当时的理解角度 / 易错点 / 自己的例子
   - 关联题号 / 错题指针
   - 上次接触日期 + 下次复习日期（见调度规则）

**沉淀原则：** 用学生自己的话和理解角度写卡片，不要照搬标准教材。卡片要能让学生一眼回到当时的理解。

### 2. 复习调度（`/积步 复习`）

扫描 `resource/积步/cards/` 与 `error_log.md`，按以下信号给出**今日复习清单**：

- 卡片「下次复习日期」≤ 今天 → 到期复习
- `error_log.md` 中 🟡待复习 且复习日期到期
- `knowledge_matrix.md` 中 ⚠️理解但不熟 且久未更新

输出：3-5 条优先级排序的复习项，每条注明**为什么复习**（到期 / 易错 / 不熟）和**怎么复习**（重看卡片的哪个点 / 重做哪道题）。

调度间隔见 `resource/积步/复习调度.md`（简化版间隔重复）。

### 3. 学习画像诊断（`/积步 画像`）

综合 `profile.md` + `knowledge_matrix.md` + `error_log.md` + `contest_log.md`，输出一份诊断报告：

- **能力雷达**：按阶段（入门/基础/提高/进阶/冲刺）统计掌握比例
- **薄弱集群**：哪几个知识点反复出错或长期不熟，可能是同一底层概念缺位
- **成长趋势**：最近 N 次做题/比赛的正确率与心态走向
- **下一步建议**：补哪个基础、练哪个模板、安排一次什么难度的题

诊断要**说人话、给具体动作**，不是"建议加强练习"这种废话。

### 4. 搜索（`/积步 搜索 <关键词>`）

在 `resource/积步/cards/` + 各记忆文件中检索关键词，返回相关卡片与历史记录条目。用于"之前讲过的某个点，学生忘了，快速翻出来"。

## 命令一览

| 命令 | 作用 |
|------|------|
| `/积步 沉淀` | 提炼当前对话知识点，更新矩阵+生成卡片 |
| `/积步 复习` | 给出今日复习清单 |
| `/积步 画像` | 生成学习画像诊断报告 |
| `/积步 搜索 <词>` | 检索知识点卡片与历史记忆 |

## 自动触发时机

非命令调用时，积步在以下时刻**主动**出手：

- 学生刚理解一个新知识点 / 做对相关题 → 提示是否沉淀（或直接沉淀）
- 学生在同一知识点第二次出错 → 主动建议纳入复习队列
- 长会话涉及多个知识点 → 收尾时建议一次批量沉淀

主动出手要**简短**：一句话提示 + 行动，不要长篇。

## 风格

遵循 `CLAUDE.md` 说话风格。沉淀和诊断是工具活儿，干净利落；但涉及"为什么复习""下一步建议"时，保持教练的温度和具体性。
